# HR-система управления сотрудниками (КЭТ им. Ф.В. Чижова)

Данный проект представляет собой **полностековое веб-приложение** для автоматизации работы отдела кадров. Разработан в рамках курсовой работы по дисциплине «Проектирование и дизайн информационных систем».

## Функциональность

- **Просмотр списка сотрудников** – таблица с ФИО, должностью, отделом.
- **Фильтрация по отделам** – динамическое обновление списка.
- **Добавление сотрудника** – форма с выбором должности и отдела.
- **Редактирование и удаление** – модальное окно, подтверждение.
- **Учёт отпусков** – создание заявок, согласование, статусы.
- **Управление вакансиями** – открытие/закрытие, учёт кандидатов.
- **Отчёты** – численность, статистика отпусков, текучесть.
- **Аутентификация** – JWT, роли (сотрудник, руководитель, HR).

## Технологии

### Бэкенд
- **Python 3.11+**  
- **FastAPI** – REST API  
- **SQLAlchemy** – ORM  
- **SQLite** – база данных (по умолчанию)  
- **JWT** – аутентификация  
- **Uvicorn** – ASGI-сервер  

### Фронтенд
- **React 18** – UI  
- **Vite** – сборка  
- **Axios** – HTTP-запросы  
- **React Toastify** – уведомления  
- **CSS** – адаптивная стилизация

## Запуск проекта (локально)

### Требования
- Python 3.11+ (с `pip`)  
- Node.js 18+ (с `npm`)

### 1. Клонирование репозитория
```bash
git clone https://github.com/RomaSesh/ket-hr-frontend.git
cd ket-hr-frontend
```
### 2. Запуск бэкенда (FastAPI)
```bash
cd backend
python -m venv venv
```
# Активировать виртуальное окружение:
# Windows:
venv\Scripts\activate
# Mac/Linux:
source venv/bin/activate

# Установить зависимости
```pip install -r requirements.txt```
Создайте файл .env в папке backend (необязательно, для SQLite можно пропустить):

```ini
DATABASE_URL=sqlite:///./hr.db
SECRET_KEY=supersecretkey
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=30
```

### Запустите сервер:

```bash
uvicorn main:app --reload --port 8000
API будет доступно: http://localhost:8000
Swagger документация: http://localhost:8000/docs
```
### 3. Заполнение базы данных тестовыми данными (опционально)
Остановите сервер (Ctrl+C) и выполните:

```bash
python seed.py
```
Скрипт создаст отделы, должности и тестового пользователя admin с паролем admin123.
После этого снова запустите uvicorn main:app --reload.


### 4. Запуск фронтенда (React)
В новом терминале (не закрывая бэкенд):

```bash
cd frontend
npm install
npm run dev
```
Приложение откроется по адресу: http://localhost:5173

### ✅ Проверка работы
Перейдите на http://localhost:5173 – должно перенаправить на /login.

Войдите с учётными данными admin / admin123 (или зарегистрируйте нового пользователя).

В таблице сотрудников появятся данные (если выполняли seed.py).

Попробуйте добавить, отредактировать, удалить сотрудника.

### 🔧 Возможные проблемы и их решение
Ошибка импорта при запуске бэкенда
Убедитесь, что активировано виртуальное окружение и выполнено pip install -r requirements.txt.
Если ошибка ImportError с относительными импортами, проверьте, что все файлы .py лежат в правильных местах (см. структуру репозитория).

CORS ошибка в браузере
Убедитесь, что в backend/main.py в allow_origins добавлен адрес фронтенда: "http://localhost:5173".
Если используете другой порт – измените.

Ошибка 422 при добавлении сотрудника
Проверьте, что в таблицах departments и positions есть записи. Запустите python seed.py для заполнения.

Порт 8000 или 5173 уже занят
Завершите процесс, использующий порт, или измените порт в командах (--port 8001 для бэкенда, --port 5174 для фронтенда) и обновите CORS.
